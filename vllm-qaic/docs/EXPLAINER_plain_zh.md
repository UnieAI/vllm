## 1. 先懂:大模型是怎么「吐字」的

大模型生成文字,是**一个字一个字挤**的(严格说是 token,一个 token 约等于一个词
的一部分,这里就当成「字」)。每挤一个字,都要让卡算一次。

```
你问:今天天气怎么样?
模型:今 → 天 → 真 → 不 → 错 → 。      ← 算了 6 次,才出 6 个字
```

**问题:每次只出一个字,卡跑一趟只赚一个字,慢。**

---

## 2. 加速技巧:投机解码(speculative decoding)

核心想法:**先便宜地「猜」接下来几个字,再让模型一次性「批改」这些猜测。**
猜对的就一起收下,一趟卡就能吐好几个字。

打个比方 —— 像**学生抢答 + 老师一次批改**:

- 学生(便宜的猜测器)抢答:接下来 3 个字我猜是 `真`、`不`、`错`。
- 老师(真正的大模型)看一眼,一次批改这 3 个:
  - 第 1 个该是 `真` ✓ 收下
  - 第 2 个该是 `不` ✓ 收下
  - 第 3 个老师觉得该是 `好`,不是 `错` ✗ → 把第 3 个改成 `好`,后面作废
- 结果:这一趟卡,吐出了 `真 不 好` **三个字**(而不是一个)。

**关键:结果和「老老实实一个一个出」完全一样,只是更快。** 不会让模型变笨。

---

## 3. 我们用的是最便宜的猜法:ngram(看历史重复)

「学生」怎么猜?最便宜的猜法叫 **ngram / prompt-lookup**:
**在已经出现过的文字里找重复,重复过的后面跟了什么,就猜它会再来一次。**

例子:翻译/复述任务里文字常重复。
```
已经生成过:... the cat sat on the mat .
现在又走到:... the cat sat on the ___
```
ngram 一看:上次「the cat sat on the」后面是「mat .」→ 那就猜接下来是 `mat`、`.`。
不用第二个模型,零成本。

---

## 4. 为什么在高通卡上,这事得「我们」来做

投机解码里有一步叫「**批改/验收**」(术语:rejection sampling,拒绝采样)——
就是上面老师批改那一步,决定每个猜测收不收。

麻烦在这:**vLLM 自带的那个「批改器」是写给 GPU 用的**(用一个叫 Triton 的
GPU 专用工具)。而:

- 高通的卡**不是 GPU**;
- 跑 vLLM 主程序的那台主机是普通 **CPU**,上面**可能根本没装 Triton**。

所以自带的批改器**在高通这边跑不起来**。没有批改器,整个加速就无法开启 ——
**这正是高通当初把这个功能直接关掉的原因**(代码里写死「暂不支持」)。

> 我们做的:**用普通 CPU 代码,把这个「批改器」重写了一遍**,数学规则和原版
> 一模一样(该按概率收的按概率收、收不下就换一个),所以结果不变,只是能在
> 高通的 CPU 主机上跑了。

## 5. 我们具体做了什么 —— 两大块

### 块 A:移植升級工程(工作主体)

「移植升級」= 把一套软件从一个版本/环境,搬到另一个版本/环境,并让它重新跑通。
这中间墙、水电(依赖、接口、内部结构)都要重新对位,**不是复制粘贴**。我们做了四步:

| 步骤 | 做了什么 | 大白话类比 |
|---|---|---|
| **A1 整合到 v0.10.1 基线** | 把高通的 QAIC patch 对齐到开源 vLLM **v0.10.1**,让它能装上、依赖对得上、跑得起来 | 把别人写给旧户型的装修方案,改到你这套新户型上 |
| **A2 V1 引擎适配 + 产品化** | vLLM 有新旧两套执行引擎。**官方高通示范用的是旧的 V0**;我们把它弄到**新的 V1 引擎**上,并产品化(你们生产就是 `VLLM_USE_V1=1`) | V1 更快但内部完全不同 —— 等于**换发动机**还要让车照常上路 |
| **A3 QEfficient/SDK 版本对接** | 对接 efficient-transformers **v1.21** + Cloud AI SDK **1.21**,把「编译模型 → 在卡上跑」整条链路打通、版本对齐 | 让新的编译工具和新的驱动彼此认得、不打架 |
| **A4 迁移到 vLLM 0.2x(进行中)** | 现在再往更新的 vLLM(0.21+)搬 —— 引擎又被大改,是更大的一次升级 | 再换一次更新的发动机 |

> **A 这块是工程主体。** 这些 qaic 文件的代码出处是高通(标「机密专有」)/ 开源 vLLM,
> 但**「让它在 v0.10.1 上、在 V1 引擎上、在新 SDK 上跑通,并往 0.2x 迁移」这部分整合
> 与升級的工程劳动,是 UnieAI 做的**。(代码出处 ≠ 整合工作;两者要分开看。)

### 块 B:ngram 投机解码优化(在 A 的底座上加的一个优化项)

| | 之前(高通出货) | 之后(我们加完) |
|---|---|---|
| **开关** | 一碰投机解码就报错「暂不支持」 | 打开,且限定用最稳的 ngram 猜法 |
| **怎么喂给卡** | 一次只验 1 个字 | 把「上一个字 + 最多 N 个猜测」打包,**一趟卡验 N+1 个** |
| **批改器** | 用 GPU 的 Triton(高通跑不了) | **我们自己写的 CPU 批改器**(7 个小函数) |

外加一处小改:让卡上的模型**一次多输出 N+1 行「打分」**(不然没东西可批改)。

> **B 这块是我们原创的代码**:约 **250 行**,集中在一个文件里 + 一份设计文档。
> 它是 speculative decode 的**一个优化项**,不是我们工作的全部。

## 6. 一个完整的小例子(从头走一遍)

设定:已经生成的文字里出现过 `the cat sat on the mat . The end`,现在又走到了
`... the cat sat on the`,且我们设 N=3(一次猜 3 个)。

```
第 1 步【猜】 ngram 回看历史:「the cat sat on the」后面跟过「mat . The」
            → 猜测 = [mat, ., The]

第 2 步【打包】 把 [上一个已确定的字] + [mat, ., The] 打成一批,一次送进高通卡
            （这就是「2D 打包」：一行装这一个请求的 N+1 个位置）

第 3 步【卡算】 卡一趟算完,给回每个位置的「打分」（每个字有多少分）

第 4 步【我们的 CPU 批改器验收】
            位置1 模型最高分 = mat，猜的也是 mat ✓ 收下
            位置2 模型最高分 = . ，猜的也是 .   ✓ 收下
            位置3 模型最高分 = She，猜的是 The   ✗ 拒绝 → 改成 She，后面作废

第 5 步【产出】 这一趟卡吐出 [mat, ., She]  → 3 个字（而不是 1 个）
```

如果三个全猜中,还会**额外白送一个字**(bonus token),最多一趟出 N+1 个。
猜不中也不亏:至少出 1 个(改对的那个),和不加速时一样。

## 8. 名词小抄

| 词 | 人话 |
|---|---|
| token | 一个「字」(模型读写文字的最小单位) |
| 投机解码 | 先猜几个字、再一次批改的加速法;结果不变,只更快 |
| ngram / prompt-lookup | 最便宜的猜法:重复过的文字,猜它再来一次 |
| 拒绝采样 / rejection sampler | 「批改器」:决定每个猜测收不收 |
| Triton | 一种 GPU 专用编程工具;vLLM 自带批改器要用它,高通主机可能没有 |
| QPC | 高通卡上能跑的、编译好的模型 |
| logits / 打分 | 模型对「下一个字该是谁」给出的分数 |
| QAIC / Cloud AI 100 | 高通的 AI 加速卡 |
